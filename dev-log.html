分析與發現
  • 原稿為 AI 產生的單檔計算器（Fenner 扭轉連桿模型）。矩形扭轉係數表與 Timoshenko 標準值完全一致、主公式 n ≥ (β/α)·Θ·G·b/(τ_allow·l) 推導正確、材料參數在合理範圍。
  • Bug①切割線 off-by-one：m 條交錯切線只形成 m−1 個串聯扭轉連桿（最外側料帶與實體板連續不扭轉），原稿畫 n−1 條實際只有 n−2 個有效連桿，成品彎不到目標角。
  • Bug②干涉角二分法失效：投影寬度 w·cosθ+t·sinθ 非單調，w≫t 時二分法跳過干涉區、把限制角錯算成 90°。
  • 修法：切線改畫 n+1 條（含格狀區兩端，恰使 W=n·(w+k) 幾何自洽）；干涉角改線性掃描找第一個違反角。
驗證
  • 預設參數手算核對全部吻合（n=23、θ=3.91°、τ=19.7 MPa、R 值）；w=10/t=3 極端案例干涉角實測 4.35°（解析解 4.38°）。部署 GitHub Pages 後線上驗證。

旋轉方塊嚴格驗證＋DXF 匯出

33360f4 · 19:35
USER旋轉方塊的方式，確定不會切斷嗎？另外有辦法輸出為 DXF 嗎？
分析與發現
  • 驗證方法升級：0.1mm 像素化＋淹水填充直接數連通元件（不靠推理）。Auxetic 預設＝1 件不切斷；h=k 時碎成 102 件；六種圖樣主體皆單一連通；長孔 69 件＝1 主體＋68 落料（挖除式正常）。
  • 幾何事實：每個角隅節點必有一條切線穿過，鉸鏈被剖成兩條韌帶，有效寬度=(h−k)/2 每側（預設只剩 0.5mm）——加 h≤k 紅旗、韌帶<0.3mm 黃旗。
  • 新功能：DXF R12 匯出（LINE 實體、CUT/OUTLINE 分層、$INSUNITS=mm，貝茲/圓弧自動展折線）。

DXF 相容性：Illustrator 開不起來

2026-07-09
USER我剛剛輸出 DXF，並透過 Illustrator 開啟，但他顯示錯誤無法開啟。
分析與修正
  • 原輸出是「極簡 DXF」（只有 HEADER＋ENTITIES）——雷射軟體通常寬容能讀，但 Illustrator 解析嚴格：缺 TABLES 區段（LTYPE/LAYER/STYLE 表）與 $ACADVER 就拒開。
  • 改為完整標準 R12 骨架：$ACADVER(AC1009)、$EXTMIN/$EXTMAX、TABLES 三表（含全部圖層定義）、BLOCKS、ENTITIES，CRLF 換行；三個匯出點共用同一 dxfDoc 產生器。
  • 驗證：四種輸出（直線/波浪貝茲展開/長孔圓弧展開/變半徑）以 ezdxf 稽核全數 0 錯誤 0 修正。
教訓：「格式驗證器通過」≠「目標軟體能開」——寬容的解析器（雷射軟體）會縱容缺漏，嚴格的（Illustrator）才逼你把規格補齊。跨軟體交換檔要以最嚴格的消費端為準。
方法論小結：①AI 產出必逐項驗證（手算核對公式、極端參數測邊界）；②「會不會切斷」這類物理問題用像素化淹水填充直接證明，不靠推理；③AI 生成器的新版本會讓舊修正回歸，合併前重查；④宣稱的文獻對照（如「Fenner 論文基準」）先查證再引用，避免循環論證。
